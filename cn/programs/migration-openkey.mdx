## 额度迁移说明（不会自动迁移）

由于 OpenKey 与 Poixe 的充值/计费体系不同，本计划不会自动把你的 OpenKey 余额迁移到 Poixe。

你可以先阅读下方差异说明与换算方式，确认理解后再联系支持团队进行人工迁移（我们会协助核对记录并做人工划转）。

## 充值与计费差异（理解换算的关键）

### OpenKey 的计费与折扣（简化理解）

- OpenKey 模型的 输入/输出 tokens [价格](https://openkey.cloud/pricing)，默认为“官方价格的 **12 倍**”
- OpenKey 额度标价：**1 RMB = 1 USD 的 OK 额度**（实际可能存在折扣，比如 0.5 RMB 买 1 USD OK 额度）
- 因为存在 12 倍倍率，你看到的 OK 额度并不等价于“官方美元余额”

**举例：老牛花 500 RMB 购买 OK 额度，实际相当于官方多少折？**

- 按照目前 OK 定价，500 RMB 可买 1000 USD OK 额度
- 1000 USD OK 额度 = 官方可消费 `1000 / 12 = 83.33 USD`
- 按照目前汇率 1 USD ≈ 7 RMB，500 RMB 为 `500 / 7 ≈ 71.43 USD`
- 所以“实际支付 / 官方可消费”：
  - `71.43 USD / 83.33 USD ≈ 0.857`
  - 即大约 **85.7% ≈ 86 折**

> 这也是为什么 OpenKey 的“倍率 + 充值折扣”叠加后，整体直觉不够清晰。

### Poixe 的计费与折扣（更直观）

- Poixe 充值余额 **直接以美元显示**：你充值多少美元，余额就显示多少美元
- 模型按官方 token 单价计算出理论消耗后，会乘以你的 [权益等级折扣](https://poixe.com/tiers/user)（level 0–3）
  - 折扣示例：`1.0`（原价）、`0.8`（8 折）等
- **余额 = 真实可用金额**（理解成本低）

## 如果我想迁移额度，怎么处理？

额度迁移需要人工处理。采取的方式是：**按你在 OpenKey 剩余额度对应的“实际付款金额”进行划转**。

也就是说，我们会把你 OpenKey 账号的剩余额度折算回“当初实际花了多少钱（RMB）”，再按汇率换算成美元，**直接充值到你的 Poixe 账号余额**。

<Callout type="info" icon="handshake">
  <b>人工迁移流程：</b>[联系支持团队](https://poixe.com/about/contact)，我们会核对你的 OpenKey 订单记录与当前剩余额度，计算可划转金额，并划转到 Poixe。
</Callout>

### 你需要提供什么

- OpenKey 账号邮箱 / 用户名
- Poixe 账号邮箱 / 用户名
- 你希望迁移额度
- 订单截图或相关记录（非必须，可加速核对）

### 我们如何计算与划转

1. **核对 OpenKey 历史订单与当前剩余额度**
2. **计算剩余额度对应的实际付款金额（RMB）**
3. **按汇率换算为美元，并划转到 Poixe 余额**
   - `Poixe 入账(USD) = 对应退款金额(RMB) / 汇率(RMB per USD)`

> 划转完成后，Poixe 的余额就是你的真实可用金额；调用消耗按 Poixe 的官方计价与等级折扣结算。

### 举个例子（老牛）

- 老牛花 **500 RMB** 购买了 **1000 USD OK 额度**
- 假设目前还剩 **600 USD OK 额度**

那么我们按剩余比例折算实际付款金额：

1) 剩余占比：`600 / 1000 = 60%`  
2) 对应实际付款：`500 RMB × 60% = 300 RMB`  
3) 按 `1 USD ≈ 7 RMB` 换算：`300 / 7 ≈ 42.86 USD`  

最终：老牛的 Poixe 账号余额将增加约 **$42.86**。
