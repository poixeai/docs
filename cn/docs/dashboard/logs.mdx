---
title: "查看日志"
---

你可以在控制台查看最近的调用日志：[https://poixe.com/log](https://poixe.com/log)

日志用于帮助你核对用量与费用、排查请求耗时与路由情况。平台会记录你每一次**成功计费**的请求，并仅保留**最近 3 天**的数据（滚动更新）。

## 日志里包含什么

每条日志通常会展示以下信息（不同模型/协议字段可能略有差异）：

* **时间**：请求发生的时间
* **令牌名称**：该请求使用的 API 令牌
* **请求模型 / 处理模型**：你请求的模型名，以及系统最终实际处理的模型标识
* **请求厂商 / 处理厂商**：是否走默认路由，以及最终由哪家供应商处理
* **Token 统计**：提示（prompt）、补全（completion）、缓存（如有）
* **费用信息**：理论消耗、实际花费、折扣系数
* **耗时**：上游耗时、系统耗时与总耗时
* **响应类型**：例如流式 / 非流式

## 如何查找一条日志

在日志页面可以按常见字段进行筛选与搜索，例如：

* 按 **令牌名称** 找到某个应用的请求记录
* 按 **模型名称** 核对某个模型的调用情况
* 通过关键字快速定位某条异常请求（例如耗时较长、费用异常等）

## 查看日志详情

点击某条日志的 **详情**，可以看到更完整的请求摘要，例如：

* 请求 ID、创建时间、响应状态
* 路由信息（默认路由/指定厂商、最终处理厂商）
* 更完整的 Token 与计费明细
* 更精确的耗时拆分（系统/上游/总耗时）
* 请求来源信息（如 IP 等）

这些信息通常足够用于：

* 对账：核对某次调用扣费是否符合预期
* 性能排查：判断耗时主要发生在上游还是网络/系统侧
* 路由确认：确认是否走了默认路由、实际由谁处理

## 保留策略说明

* 日志仅保留 **最近 3 天**
* 采用滚动策略：新的记录会持续写入，过期数据会自动移除
* 如需更长周期的审计或对账需求，建议你在业务侧自行落库保存关键字段（例如请求 ID、模型、费用、时间）
